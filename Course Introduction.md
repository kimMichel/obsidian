# Introduction

**Web Red Team Analyst (WEB-RTA**) course focuses on manual penetration testing techniques for assessing the security of web application.

# What this course is about?

Introducing to Web Applications, their commons vulnerabilities and testing guides.

How to identify, exploit, and escalate Web App misconfigurations to achieve:

- Initial Access
- Privilege Escalation
- Lateral Movement

# What this course is not about?

This is not a bug bounty course.

No 0-day / N-day exploits will be used.

Tools usage will not be covered in-depth.

Course focus in on offensive techniques, not defense or configuration best practices.

# Pre-Requisites

System with 16GB+ RAM & 256 GB SSD/HDD.

Ability to run a hypervisor on the system (Hyper-V, QEMU, VirtualBox, or VMware).

Comfortable with basic networking and HTTP concepts (requests, responses, headers, status codes, ports).

Familiarity with HTML, JavaScript, and web application architecture (client/server, cookies, sessions).

Basic Linux command-line skills (shell, editing files, process management) and file-system navigation.

Some programming experience (Python, JavaScript, or similar) sufficient to read and write small scripts.

# Practice Lab Information

Web-RTA practice lab comes in a bundled ZIP format with Dockerfile & source code.

This ZIP file can be downloaded from the CCSP portal under Lab Access section.

Further instructions for running the lab locally will be provided in the ZIP file.

# Exam Lab Information

Web-RTA comes with a 30-day exam lab.

User can start the exam lab at their own convenience and lab access will be automatically terminated after 30 days.

In order to earn the certification, student must submit 10 flags in the CCSP portal.

Students are allowed unlimited attempts to earn the certification.
